What is a Pushdown Automaton?

A Pushdown Automaton (PDA) is a finite automaton augmented with an additional stack memory. The stack provides unlimited LIFO storage, enabling PDAs to recognize a strictly larger class of languages than regular finite automata — specifically, the class of context-free languages.

💡 PDA = Finite Automaton + Unlimited Stack (LIFO). Stack = the secret superpower.

Formal Definition: M = (Q, Σ, Γ, δ, q₀, Z₀, F)

Q— finite set of states
Σ— input alphabet (symbols accepted from tape)
Γ— stack alphabet (symbols that can be on the stack)
δ— transition function: Q × (Σ∪{ε}) × Γ → 2^(Q×Γ*)
q₀— start state
Z₀— initial stack symbol (bottom-of-stack marker)
F— set of accept states
How Transitions Work
δ(q, a, A) = (p, γ) — In state q, reading 'a', stack top 'A' → go to p, replace A with γ

Each step the PDA reads one input symbol (or ε), peeks at the stack top, and chooses a transition:

  • If push=[A,B] — pop stack top, push B then A (A ends up on top)
  • If push=[] — pop stack top (POP operation, γ = ε)
  • ε-transitions let the PDA act without consuming input (nondeterminism)

By Final State
Accept if all input consumed and current state ∈ F. Stack contents irrelevant.
By Empty Stack
Accept if all input consumed and stack is empty. State irrelevant. Equivalent power.
Real-World Applications
⟨/⟩ Parsing Languages

CFGs describe syntax; PDAs parse them. Every compiler uses this.

🏷 XML/HTML Validation

Matching opening/closing tags requires stack-based memory.

🔧 Compiler Front-Ends

Syntax analysis and recursive descent parsing.

⚖ Expression Evaluation

Balanced parentheses, operator precedence, arithmetic grammars.

aⁿbⁿ — Equal number of a's and b's
The classic PDA example. Push one 'A' for each 'a' read, then pop one 'A' for each 'b'. Accept when input is consumed and stack is empty (except Z₀).
ww^R — Palindromes over {a, b}
Push the first half onto the stack, nondeterministically guess the midpoint, then pop matching symbols for the second half.
Balanced Brackets — { ( ) [ ] { } }
Push every opening bracket; on a closing bracket, verify and pop the matching opener from the stack top.
⚠ aⁿbⁿcⁿ — NOT a Context-Free Language
PDAs cannot recognize this language. A PDA needs two independent counters, but the stack only provides one LIFO memory. This demonstrates the fundamental limits of pushdown automata — you need a Turing Machine for this.
