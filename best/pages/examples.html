01
✓ Context-Free
aⁿbⁿ Language
L = { aⁿbⁿ | n ≥ 1 }. Equal a's then equal b's. The PDA uses the stack as a counter: one push per 'a', one pop per 'b'. When the stack returns to just Z₀, the counts are equal.

How the PDA handles aⁿbⁿ

q0 Reading first 'a': push A on top of Z₀ → stack=[A,Z]. Move to q1.
q1 Each 'a': push A. Stack grows one level per 'a'. Depth = n.
q1 Each 'b': pop one A. Stack shrinks. Transitions stay in q1.
q1→q2 When stack top = Z (all A's popped) and input ends: ε-move to q2 (accept state).
REJECT If 'b' arrives when stack top is Z, or input ends with A's still on stack → no valid transition → reject.
✓ ACCEPTED

q0 ─ a,Z/AZ → q1 stack:[A,Z]
q1 ─ b,A/ε → q1 stack:[Z]
q1 ─ ε,Z/Z → q2 ACCEPT ✓

ww^R Palindromes
L = { ww^R | w ∈ {a,b}* }. Even-length strings that are their own reverse. The PDA nondeterministically guesses the midpoint: push the first half, ε-transition at the middle, then pop while matching the reversed second half.

How the PDA handles ww^R

q0 Reads and pushes every symbol (a→A, b→B). Stays in q0.
q0→q1 At any point: ε-transition to q1 (nondeterministic guess of midpoint). No input consumed, no stack change.
q1 For each remaining input symbol, must match the stack top. Matching = pop. Mismatch = no rule = reject.
q1→q2 When all input consumed and stack top = Z: ε-move to q2 (accept).
✓ ACCEPTED
abba
Reverse of "ab" is "ba". Classic 4-char palindrome.
Push A,B → stack:[B,A,Z]
Guess mid (ε,q0→q1)
Pop B for 'b', A for 'a' → ACCEPT ✓

Balanced Brackets
L = { w | w is a balanced sequence of ( ) [ ] { } }. Every compiler uses this exact algorithm. The PDA pushes each opener and verifies/pops on each closer. A mismatch (e.g. '(' followed by ']') immediately rejects.

How the PDA handles brackets

OPEN ( → push '(' on stack. [ → push '['. { → push '{'. Stay in q0.
CLOSE ) requires stack top = '('. ] requires '['. } requires '{'. Pop on match.
MISMATCH Close bracket but wrong type on stack top → no matching rule → REJECT.
ACCEPT Input ends, stack top = Z → ε-move to q1 (accept). All brackets matched.
REJECT Input ends but opener(s) still on stack → unclosed brackets → REJECT.
✓ ACCEPTED

Push ( → stack:[(,Z]
Pop ( for ')' → stack:[Z]
ε → q1 → ACCEPT ✓

✗ NOT Context-Free
aⁿbⁿcⁿ — PDA Limit
L = { aⁿbⁿcⁿ | n ≥ 1 }. Provably NOT a context-free language. No PDA can recognize it. Requires two independent counters — a PDA provides only one LIFO stack.
⚠ Why PDAs Cannot Recognize aⁿbⁿcⁿ
A PDA has exactly one stack. After using it to count n a's and verify n b's (by depleting the stack), there is no memory left to verify that exactly n c's follow. The stack is already empty.

Pumping Lemma proof: Take s = aᵖbᵖcᵖ (|s| = 3p ≥ p). Any decomposition s = uvxyz with |vxy| ≤ p means vxy lies within at most 2 symbol types. Pumping i = 2 increases those 2 types' counts while leaving the third unchanged — breaking the equal-count condition. Since no valid pumping exists, the language is not a CFL.

What CAN recognize it: A linear-bounded automaton (Type 1, context-sensitive) or a full Turing Machine can count all three independently.
