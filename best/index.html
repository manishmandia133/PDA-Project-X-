01
01 — Formal Model

FORMAL DEFINITION

A Pushdown Automaton is formally defined as a 7-tuple. It generalizes a finite automaton by adding an auxiliary stack that provides unbounded but LIFO-restricted memory. Unlike a DFA/NFA which can only remember a finite amount, the PDA's stack can grow arbitrarily, enabling it to recognize context-free languages.

M = (Q, Σ, Γ, δ, q₀, Z₀, F)
Q
Finite set of states. The PDA exists in exactly one state at any moment.
Σ
Input alphabet. Finite set of symbols read from the input tape left-to-right.
Γ
Stack alphabet. May differ from Σ. Symbols that can appear on the stack (e.g. A, B, Z₀).
δ
Transition function: Q × (Σ ∪ {ε}) × Γ → 2^(Q×Γ*). May be nondeterministic (return a set of choices).
q₀
Start state. q₀ ∈ Q. Computation begins here.
Z₀
Initial stack symbol. Z₀ ∈ Γ. Placed at the bottom of the stack before any input is read. Acts as a sentinel / bottom marker.
F
Set of accept states. F ⊆ Q. Used in acceptance by final state.
Chomsky Hierarchy

PDA IN CONTEXT

Noam Chomsky's hierarchy classifies formal grammars by their expressive power. PDAs correspond exactly to Type 2 — context-free grammars. They are strictly more powerful than finite automata but strictly less powerful than Turing Machines.

T0

T1

Linear-bounded automaton · LBA
T2

T3

02 — Transitions & Acceptance

HOW TRANSITIONS WORK

Each computation step of a PDA is determined by three inputs: the current state, the next input symbol (or ε), and the current stack top. The transition function maps these to a new configuration.

📖
① Read Input
Consume one symbol from the input tape (or use an ε-transition, which consumes nothing). The read head moves strictly left to right and cannot go back.
01
📦
② Inspect Stack
Look at the top of the stack (top symbol only is visible). The current stack top determines which transitions are applicable — it is part of the transition guard.
02
🔄
③ Transition
Pop the stack top, push the new string γ onto the stack. If γ = ε that's a pure pop. Move to the new state. Repeat until input exhausted.
03

Transition Notation: δ(q, a, A) = (p, γ)

δ(q, a, A) = (p, γ)
→Current state: q
→Input symbol consumed: 'a' (or ε for no consumption)
→Stack top popped: 'A'
→Move to new state: p
→Push string γ onto stack. If γ = AB, then B is pushed first, A ends up on top. If γ = ε, stack shrinks (pure pop).
Acceptance Modes — Both Have Equal Power
By Final State
After all input is read, if the current state q ∈ F the string is accepted. Stack contents are irrelevant. Commonly used in simulator implementations.
By Empty Stack
After all input is read, if the stack is completely empty the string is accepted. The state is irrelevant. Formally equivalent — any PDA can be converted between the two modes.

03 — CFL Properties

PUMPING LEMMA

The Pumping Lemma for CFLs is used to prove that certain languages are NOT context-free — and therefore cannot be recognized by any PDA. If a language fails the lemma, no PDA can recognize it.

Statement: For every CFL L, there exists a pumping length p such that any string s ∈ L with |s| ≥ p can be split into five parts: s = uvxyz, satisfying all three conditions:

①
|vy| ≥ 1 — at least one of v or y is non-empty
②
|vxy| ≤ p — the pumpable middle is bounded by p
③
uvⁱxyⁱz ∈ L for all i ≥ 0 — pumping v and y any number of times keeps the string in L
—

WHY aⁿbⁿcⁿ FAILS

The language L = {aⁿbⁿcⁿ | n ≥ 1} is the classic example of a language that is NOT context-free. Any pumping attempt breaks the equal-count condition.

Why a PDA cannot count three things: A PDA has one stack — providing one LIFO counter. To verify aⁿbⁿcⁿ you need to count n a's, then verify n b's (depleting the counter), then verify n c's (requiring a second fresh counter). Once the stack is depleted matching b's, no information about n remains for the c's.

Pumping Lemma proof sketch: Take s = aᵖbᵖcᵖ. Any split uvxyz where |vxy| ≤ p means vxy spans at most two symbol types. Pumping v and y (i=2) increases at most two symbol counts — breaking the equality aⁿbⁿcⁿ.

04 — Applications

REAL-WORLD APPLICATIONS

Context-free languages and PDAs are not academic curiosities — they underpin nearly every programming tool you use daily.

⟨/⟩
Compiler Front-Ends
Every language parser (C, Python, JavaScript, Rust) uses a PDA-equivalent algorithm. The production rules of a programming language grammar are directly a CFG parsed by an LALR or LL PDA.
gcc · LLVM · V8
🏷
XML & HTML Parsing
Matching open/close tags (e.g. <div>…</div>) is exactly the balanced-brackets problem. Every browser's DOM parser and XML processor implements a PDA internally to verify nesting.
WebKit · Blink · libxml2
🔧
Programming Language Design
Language designers specify syntax via Context-Free Grammars and use parser generators (YACC, ANTLR, Bison) that automatically synthesize the PDA from grammar rules.
ANTLR · Bison · YACC
⚖
Expression Evaluation
Arithmetic expressions like (a + b) * (c - d) have nested structure. Shunting-Yard algorithm and recursive descent parsers are PDA implementations that handle operator precedence.
SQL · Math parsers · REPL
🛡
Natural Language Processing
Phrase-structure grammars in NLP (Chomsky's own contribution) use CFGs to parse sentences. CYK and Earley algorithms are PDA variants adapted for ambiguous natural language.
NLP · CYK · Earley
📦
Serialization Formats
JSON, YAML, and TOML parsers validate nested structure using PDA-equivalent logic. The recursive nesting of objects and arrays is a direct CFL application.
JSON · YAML · TOML
