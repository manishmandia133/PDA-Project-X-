// WHAT IS A PUSHDOWN AUTOMATON?

A Pushdown Automaton (PDA) is a finite automaton enhanced with an auxiliary stack memory. The stack provides unbounded, LIFO storage — enabling PDAs to recognize a strictly larger class of languages than regular finite automata: the class of context-free languages (CFL).

💡 PDA = Finite Automaton + Stack  |  Stack = unlimited LIFO memory
Formal Definition: M = (Q, Σ, Γ, δ, q₀, Z₀, F)
Q— finite set of states
Σ— input alphabet (symbols read from tape)
Γ— stack alphabet (symbols pushed/popped)
δ— transition function: Q × (Σ∪{ε}) × Γ → 2^(Q×Γ*)
q₀— initial state
Z₀— initial stack symbol (bottom-of-stack marker)
F— set of final/accept states
// HOW TRANSITIONS WORK
δ(q, a, A) = (p, γ)  →  In state q, reading 'a', stack top 'A':
→ move to state p, replace A with γ on the stack

On each step the PDA:

  • Reads one input symbol (or ε — no input consumed)
  • Peeks at the current stack top
  • Chooses a rule: push γ replaces top, push ε pops the top
  • ε-transitions allow nondeterminism — the simulator takes the first valid match
By Final State
Accept when all input consumed and current state ∈ F. Stack content irrelevant.
By Empty Stack
Accept when all input consumed and stack is empty. State irrelevant. Equivalent power.
// REAL-WORLD APPLICATIONS
⟨/⟩ Parser Design
CFGs describe syntax; PDAs parse them. Every compiler front-end relies on this.
🏷 XML/HTML Validation
Matching open & close tags requires exactly the stack memory a PDA provides.
🔧 Compiler Analysis
Recursive descent + LL/LR parsers implement PDA transitions explicitly.
⚖ Expression Eval
Balanced parentheses, operator precedence, and arithmetic grammars.
// aⁿbⁿ — Equal a's and b's
The classic PDA example. Push one 'A' for every 'a' read, then pop one 'A' for every 'b'. Accept when input is consumed and only Z₀ remains on the stack.
// ww^R — Even-length Palindromes
Push the first half onto the stack, nondeterministically guess the midpoint, then pop matching symbols for the reversed second half.
// Balanced Brackets — ( ) [ ] { }
Push every opening bracket. On seeing a close bracket, verify the matching open bracket is on top of the stack, then pop it. Accept when input and stack are empty.
// aⁿbⁿcⁿ — NOT Context-Free ⚠
PDAs cannot recognize this language. It would need two independent counters, but the stack only provides one LIFO memory. This demonstrates the fundamental limit of PDAs — a Turing Machine would be required. The simulator will always reject.
